AURA FX - MYSQL WORKBENCH CONNECTION SETUP:

IMPORTANT: This is for the NEW AURA FX database, separate from "theglitches"

1. Open MySQL Workbench
2. Click "+" to create new connection
3. Enter these details (get the actual values from Railway):

Connection Name: AURA FX Railway MySQL
Hostname: <Get from Railway MySQL service - MYSQL_HOST variable>
Port: <Get from Railway MySQL service - MYSQL_PORT variable (usually 3306)>
Username: <Get from Railway MySQL service - MYSQL_USER variable (usually root)>
Password: <Get from Railway MySQL service - MYSQL_PASSWORD variable>
Default Schema: <Get from Railway MySQL service - MYSQL_DATABASE variable (usually railway)>

4. Click "Test Connection" - should work!
5. Click "OK" to save
6. Connect to the database

TO GET RAILWAY CREDENTIALS:
1. Go to Railway.app
2. Open your AURA FX project
3. Click on the MySQL service
4. Go to "Variables" tab
5. Copy the values for:
   - MYSQL_HOST
   - MYSQL_PORT
   - MYSQL_USER
   - MYSQL_PASSWORD
   - MYSQL_DATABASE

TO IMPORT YOUR SQL FILES:
1. Connect to AURA FX Railway MySQL
2. Go to Server â†’ Data Import
3. Select "Import from Self-Contained File"
4. Choose each SQL file
5. Select target database (usually "railway")
6. Click "Start Import"

OR use the import script:
Run: import_railway.bat
Enter the values from Railway MySQL service variables

VERCEL ENVIRONMENT VARIABLES NEEDED:
After setting up Railway MySQL, add these to Vercel:
- MYSQL_HOST=<from Railway>
- MYSQL_PORT=<from Railway>
- MYSQL_USER=<from Railway>
- MYSQL_PASSWORD=<from Railway>
- MYSQL_DATABASE=<from Railway>
- MYSQL_SSL=true

See RAILWAY_DATABASE_SETUP.md for detailed instructions.
